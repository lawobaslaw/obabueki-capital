# Obabueki Capital – Project Charter

## Vision
Build **Obabueki Capital**, a personal multi-platform portfolio management application that I use to track investments across multiple brokers, exchanges, and asset classes while strengthening my software engineering and DevOps skills.

---

# Goals

## Primary Goal
Obtain a **Junior DevOps / Cloud Engineer** role by demonstrating practical experience building, testing, deploying, and operating a real application.

## Secondary Goal
Build a portfolio management application that I genuinely use to manage my own investments.

## Long-Term Goal
Support my future automotive business by developing software engineering, DevOps, and AI skills while balancing my IMI mechanics course.

---

# Two-Track Roadmap

## Product Track
- Authentication
- Portfolios
- Accounts
- Transactions
- Holdings
- Portfolio Valuation
- Dashboard
- Dividend Support
- Import (CSV/APIs)

**Question:** Does this make the application more useful?

## DevOps Track

- GitHub Actions
- Azure Deployment
- Infrastructure as Code (Terraform/Bicep)
- Monitoring
- Logging
- Secrets Management

**Question:** Does this improve my DevOps portfolio?

---

# Engineering Workflow

1. Product Requirement
2. Design Review
3. Implementation
4. AI Review (Work Mode / Codex)
5. Pull Request Review
6. Merge
7. CI/CD Validation
8. Deployment

---

# Sprint Principles

- Build the MVP first.
- Avoid unnecessary features.
- Deliver one Product improvement each sprint.
- Deliver one DevOps improvement each sprint.
- AI assists; I remain the engineer.
- Prefer simple, maintainable solutions.

---

# Guiding Questions

Before starting work ask:

- Does this make the app more useful?
- Does this improve my DevOps portfolio?
- Is this required for the MVP?
- Can this be completed this sprint?

If not, move it to the backlog.
